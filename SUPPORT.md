# Support

This page explains where to ask for help and where to report issues.

## Questions and Usage Help

For general questions, setup help, deployment problems, and usage discussions, use GitHub Issues.

Before opening an issue:

- Check the README and existing issues
- Include the InsightFlare version or commit you are using
- Include your deployment method and relevant Cloudflare environment details
- Include logs or screenshots when they help explain the problem

## Bug Reports

Use the bug report issue template for reproducible problems.

A useful bug report includes:

- What happened
- What you expected to happen
- Steps to reproduce
- Version, commit, or deployment method
- Browser and Cloudflare environment when relevant

## Feature Requests

Use the feature request issue template for product ideas, workflow improvements, and integration requests.

Describe the problem first, then the proposed solution.

## Security Issues

Do not open public issues for security vulnerabilities.

Report security issues by following [SECURITY.md](SECURITY.md).

## Commercial Support

Commercial support and hosted-service support channels may be added later as the InsightFlare cloud platform matures.
